Privacy Policy — SF Peek

Effective date: 2026-03-16

Overview

SF Peek is a Chrome Extension designed for Salesforce administrators and developers. It operates entirely within your browser and communicates exclusively with Salesforce servers you are already logged in to. We collect no personal data, send no data to external servers, and include no analytics or tracking of any kind.

Data We Access

  • Salesforce session cookie (sid): Read from your browser's cookie store to authenticate API requests. This value is sent only to *.salesforce.com and *.force.com — never to any third-party server.
  • Active tab URL: Read to auto-detect the current Salesforce record so the extension can pre-fill the object name. The URL is processed locally and never transmitted.

Data We Store Locally

The extension uses chrome.storage.local (stored only on your device) for:

  • Salesforce domain / session metadata (to avoid re-reading the cookie on every popup open)
  • SOQL query history (for the SOQL Runner tab)

This data never leaves your device and can be cleared at any time via Chrome's "Clear browsing data" or by uninstalling the extension.

Third-Party Libraries

  • JSZip (MIT License) — used locally to create ZIP archives for file downloads. The library runs entirely in your browser; no data is sent to any JSZip server.
  • Tailwind CSS — compiled into the extension's CSS file at build time. No runtime CDN requests are made.

No External Communication

SF Peek makes no requests to any server other than the Salesforce org you are currently logged in to. There is no telemetry, no crash reporting, no usage analytics, and no advertising network integration.

Permissions Justification

  • cookies — Required to read the Salesforce sid session cookie for API authentication.
  • storage — Required to persist SOQL history and session metadata locally.
  • activeTab — Required to read the URL of the currently open Salesforce tab for auto-detection of the record/object context.
  • Host permissions (*.salesforce.com, *.force.com) — Required to call Salesforce REST and Tooling APIs on the user's org.

Children's Privacy

SF Peek is a developer tool intended for adult Salesforce professionals. It does not knowingly collect data from anyone under 13.

Changes to This Policy

If this policy is updated, the new version will be published at this URL. The effective date at the top of the page will reflect the change.

Contact

Questions or concerns? Open an issue at https://github.com/PoaPoa5/sf-quick-admin/issues.

プライバシーポリシー — SF Peek

発効日： 2026年3月16日

概要

SF Peek は、Salesforce 管理者・開発者向けの Chrome 拡張機能です。本拡張機能はブラウザ内でのみ動作し、ログイン済みの Salesforce サーバーとのみ通信します。個人情報の収集、外部サーバーへのデータ送信、アナリティクスやトラッキングは一切行いません。

アクセスするデータ

  • Salesforce セッション Cookie（sid）：API リクエストの認証に使用するため、ブラウザの Cookie ストアから読み取ります。この値は *.salesforce.com および *.force.com にのみ送信され、第三者サーバーには送信されません。
  • アクティブタブの URL：現在開いている Salesforce レコードを自動検出し、オブジェクト名を事前入力するために使用します。URL はローカルで処理され、外部には送信されません。

ローカルに保存するデータ

拡張機能はお使いのデバイスにのみ保存される chrome.storage.local を以下の目的で使用します：

  • Salesforce ドメイン・セッションのメタデータ（ポップアップを開くたびに Cookie を再読み取りしないようにするため）
  • SOQL クエリの履歴（SOQL ランナータブ用）

このデータはデバイスの外に出ることはなく、Chrome の「閲覧データを削除」または拡張機能のアンインストールによりいつでも削除できます。

使用しているサードパーティライブラリ

  • JSZip（MIT ライセンス）— ファイルダウンロード用の ZIP アーカイブ作成にブラウザ内で使用します。JSZip のサーバーにデータは送信されません。
  • Tailwind CSS — ビルド時に拡張機能の CSS ファイルにコンパイルされます。実行時に CDN へのリクエストは発生しません。

外部との通信なし

SF Peek は、現在ログインしている Salesforce 組織以外のサーバーへのリクエストを一切行いません。テレメトリ、クラッシュレポート、利用統計、広告ネットワークとの連携は一切ありません。

権限の説明

  • cookies — API 認証用に Salesforce の sid セッション Cookie を読み取るために必要です。
  • storage — SOQL 履歴とセッションメタデータをローカルに保存するために必要です。
  • activeTab — 現在開いている Salesforce タブの URL を読み取り、レコード・オブジェクトのコンテキストを自動検出するために必要です。
  • ホスト権限（*.salesforce.com、*.force.com） — ユーザーの組織の Salesforce REST API および Tooling API を呼び出すために必要です。

児童のプライバシー

SF Peek は Salesforce のプロ向けの開発ツールであり、13歳未満の方のデータは意図的に収集していません。

本ポリシーの変更

本ポリシーを更新する場合は、この URL に新しいバージョンを公開します。ページ上部の発効日が変更を反映します。

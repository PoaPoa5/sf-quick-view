# SF Peek — スクリーンショット撮影ガイド

Chrome Web Store には最大5枚のスクリーンショットを登録できます（推奨サイズ：**1280×800 または 640×400 px の PNG**）。

---

## 撮影前の準備

1. Salesforce 組織にログインした状態で Chrome を開く
2. 拡張機能を最新状態で読み込み済みであることを確認（`chrome://extensions` → 再読み込み）
3. ブラウザのズームを **100%** に設定（Ctrl+0）
4. スクリーンショットツール：Windows のスニッピングツール（Win+Shift+S）など

---

## 推奨スクリーンショット 5 枚

### Shot 1 — オブジェクト・フィールド参照タブ
**目的：** メインの機能を最初に見せる

**手順：**
1. Salesforce で任意のレコードページを開く（例：取引先）
2. 拡張機能ポップアップを開く
3. 「オブジェクト・フィールド参照」タブを選択
4. フィールド一覧が表示されている状態で撮影

**ポイント：** フィールドの API 参照名・型・ラベルが読めるように

---

### Shot 2 — SOQL ランナータブ
**目的：** クエリ実行機能をアピール

**手順：**
1. 「SOQL ランナー」タブを選択
2. `SELECT Id, Name FROM Account LIMIT 5` などのクエリを入力・実行
3. 結果テーブルが表示されている状態で撮影

**ポイント：** クエリと結果テーブルが両方見えるように

---

### Shot 3 — ER ダイアグラム（フルページ）
**目的：** 視覚的にインパクトのある機能を見せる

**手順：**
1. 「ミニ ER ダイアグラム」タブからオブジェクトを選択
2. 「フルページで開く」ボタンをクリック
3. `er.html` の全画面ダイアグラムを撮影

**ポイント：** 複数のオブジェクトノードとリレーション線が見えるように配置してから撮影

---

### Shot 4 — 自動化ロジックタブ
**目的：** トリガー・入力規則の確認機能を見せる

**手順：**
1. Apex トリガーや入力規則が設定されているオブジェクト（例：商談）を開く
2. 「自動化ロジック」タブを選択
3. ルール一覧が表示されている状態で撮影

**ポイント：** 各ロジックの種別（トリガー・入力規則など）が識別できるように

---

### Shot 5 — フィールドビューワー または ファイル検索タブ
**目的：** 残りの機能を補完

**オプション A — フィールドビューワー：**
1. Salesforce でレコードを開く
2. 「フィールドビューワー」タブを選択し、フィールド値一覧を表示して撮影

**オプション B — ファイル検索：**
1. 「ファイル検索」タブを選択
2. キーワードで検索し、結果ファイル一覧が表示された状態で撮影

## ストアへのアップロード

1. [Chrome Web Store デベロッパーダッシュボード](https://chrome.google.com/webstore/devconsole) にアクセス
2. 新しいアイテムを作成
3. `sf-peek-v1.0.0.zip`（`npm run pack` で生成）をアップロード
4. スクリーンショット・説明文（`store/description_ja.md` の内容）を入力
5. プライバシーポリシー URL: `https://PoaPoa5.github.io/sf-quick-admin/privacy_policy.html`
6. 審査に提出

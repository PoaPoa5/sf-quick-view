# **SF Peek 追加機能要件**

現在開発中のChrome拡張機能に、以下の2つの管理者向け便利機能を追加実装してください。

既存のUI（ポップアップまたはサイドパネル）をタブ切り替えにするか、下部にエリアを設けて配置してください。

## **追加機能A: 現在レコードの「全項目」透過ビューア**

* **機能概要**: Salesforceのレコード詳細画面を開いている状態で拡張機能を起動すると、ページレイアウトに無い項目も含め、そのレコードの全データ（API参照名と値のペア）を一覧表示する。  
* **詳細要件（AIへの実装指示）**:  
  1. chrome.tabs.query で現在アクティブなタブのURLを取得する。  
  2. URLがLightningのレコードページ（例: \*/lightning/r/\[オブジェクト名\]/\[レコードID\]/view）であるか判定し、オブジェクト名とレコードIDを抽出する。  
  3. 抽出できた場合、Salesforce REST API (/services/data/vXX.X/sobjects/\[オブジェクト名\]/\[レコードID\]) を叩いてレコード情報を取得する。  
  4. 取得したJSONデータを、テーブル形式（左列：API参照名、右列：値）で一覧表示する。  
  5. URLからレコードIDが判定できない画面にいる場合は、「Salesforceのレコード画面を開いてください」というメッセージを表示する。

## **追加機能B: 管理者向け 1クリック・ショートカット集**

* **機能概要**: よく使う設定画面へ一瞬で飛べるリンク集を、UIの目立つ場所（ヘッダー下やフッターなど）に常に表示する。  
* **詳細要件（AIへの実装指示）**:  
  以下のLightning設定画面への相対リンクボタン（またはアイコン）を配置する。クリックすると新しいタブで開くか、現在のタブを遷移させる。  
  * オブジェクトマネージャ: /lightning/setup/ObjectManager/home  
  * ユーザー管理: /lightning/setup/ManageUsers/home  
  * フロー: /lightning/setup/Flows/home  
  * プロファイル: /lightning/setup/EnhancedProfiles/home  
  * 開発者コンソール: (既存セッションを利用して /apex/debug/DevConsole 等を開く)
